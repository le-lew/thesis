# 基于容器编排的微服务弹性伸缩策略研究与实践（中期版摘要）

> 说明：论文正式版本以 `docs/paper/刘乐-毕业论文终稿.docx` 为准。本 Markdown 文件保留中期阶段的文字摘要，便于代码检查和过程材料查看。

## 摘要

随着云原生架构的发展，微服务系统在可扩展性和可维护性方面具有明显优势，但也带来了资源管理复杂、负载波动响应困难等问题。本课题围绕 Kubernetes 场景下的微服务弹性伸缩展开研究，重点分析传统单指标伸缩策略在响应滞后、资源利用率不足和伸缩稳定性方面的局限，构建可复现实验平台，并通过对照实验验证不同伸缩策略的效果。

中期阶段已经完成 Kubernetes 实验环境搭建、基础服务部署、CPU-HPA 接入、监控链路调试和 Locust 压测流程。后续阶段继续完成多指标 HPA、HPA+VPA 协同策略和正式对照实验，并将实验结果写入论文终稿。

关键词：微服务；Kubernetes；弹性伸缩；HPA；VPA；Prometheus；Locust

## 1. 研究背景

在业务流量持续波动和成本约束增强的背景下，静态资源配置往往导致高峰期资源不足、低峰期资源浪费。Kubernetes 提供了 HPA、VPA 等自动伸缩机制，为微服务弹性治理提供了基础能力，但工程实践中仍面临指标单一、策略冲突、参数调优困难等问题。

## 2. 研究目标

本课题目标是构建微服务弹性伸缩实验平台，完成以下任务：

1. 搭建 Kubernetes 实验环境并部署示例微服务。
2. 实现固定副本、CPU-HPA、多指标 HPA、HPA+VPA 四组策略。
3. 使用 Locust 构造突发负载并采集性能与资源指标。
4. 形成可追溯的原始数据、汇总数据和论文图表。
5. 分析不同策略在性能、资源利用率、失败率和成本代理指标上的差异。

## 3. 系统设计

系统由五个部分组成：

- 示例微服务：`services/sample-api/app.py`，提供业务接口和 Prometheus 指标。
- 部署配置：`deploy/base/`，定义 Namespace、Deployment 和 Service。
- 伸缩策略：`deploy/hpa/` 与 `deploy/vpa/`，定义 HPA/VPA 对照组。
- 监控链路：`deploy/monitoring/`，通过 Prometheus 与 Adapter 暴露自定义 QPS 指标。
- 实验流水线：`scripts/run_experiments.ps1`、`results/aggregate_experiment.py`、`results/plot_results.py`。

## 4. 阶段结果

正式实验阶段已经完成 `burst` 场景下四组策略各 3 次实验，共 12 条有效记录。结果显示，`hpa_vpa` 在 P95 延迟、QPS 和失败率方面整体表现较好；固定副本策略在突发负载下失败率较高；多指标 HPA 能比单 CPU HPA 更充分反映业务流量压力。

## 5. 结论

本项目已经具备“可运行、可压测、可采集、可分析、可答辩”的实验闭环。论文和答辩应重点强调：本系统是面向伸缩策略研究的实验评估平台，实验结论由 manifest、raw CSV、summary CSV 和 figures 共同支撑，目标是保证流程可重跑、数据可追溯、趋势可复现。
