# 中期检查表填写参考

## 一、论文（设计）进展

本课题围绕“基于容器编排的微服务弹性伸缩策略研究与实践”开展。前期已完成选题论证、开题报告和任务书要求的基础工作，并搭建了基于 Minikube 的 Kubernetes 实验环境。

当前已经完成 `sample-api` 示例微服务、Kubernetes Deployment/Service、基础 HPA、监控组件和 Locust 压测脚本。项目形成了“服务部署 - 监控采集 - 负载压测 - 结果归档 - 数据分析”的实验闭环，能够支撑后续多策略对比实验。

中期阶段之后，项目进一步完成了四组策略的正式对照实验，包括固定副本、CPU-HPA、多指标 HPA、HPA+VPA 协同，并生成了原始数据、汇总数据和论文图表。

## 二、存在问题及整改措施

当前主要问题包括：

1. Kubernetes 实验环境依赖较多，复现实验前需要准备 Minikube、kubectl、Docker、metrics-server、Prometheus Adapter 和 VPA 组件。
2. 不同机器上的资源状态会影响实验数值，因此结果说明应强调趋势复现和过程可追溯。
3. 答辩时需要避免把项目描述为普通业务系统，应突出“实验平台”和“策略评估系统”的定位。

整改措施：

1. 在 README 中保留完整复现步骤和故障排查说明。
2. 在 `results/raw/` 中保留每次实验的 manifest、Locust CSV、HPA/VPA 输出、events 和 top 采样日志。
3. 使用 `results/aggregate_experiment.py` 对数据进行自动校验，避免缺列、空值、run 数不足和 VPA 降级数据混入正式结论。
4. 准备答辩学习材料、问答速查和系统检查清单，提高现场表达流畅度。
