## 一、论文（设计）进展（包括前期完成情况）

本课题围绕“基于容器编排的微服务弹性伸缩策略研究与实践”开展，前期已完成选题论证、开题报告和任务书要求的基础工作。当前已完成实验环境搭建与初步验证：基于 Minikube 搭建 Kubernetes 集群，完成 `sample-api` 服务部署，配置基础 HPA（CPU 指标）和监控组件。已实现基于 Locust 的压测脚本与自动化压测流程，完成两轮实验数据采集并形成原始数据文件。

目前已初步形成项目工程骨架（部署模板、压测脚本、结果目录与可视化脚本），并完成中期版论文初稿框架与核心章节撰写。整体进度与开题计划基本一致，已具备进入“多指标 HPA、HPA-VPA 协同与对比实验”阶段的条件。

## 二、存在问题及整改措施

当前主要问题包括：  
1. 监控链路在不同环境下稳定性不一致，早期存在 Metrics API 不可用问题。  
2. 当前完成的是基础 HPA 验证，多指标 HPA 与 HPA-VPA 协同实验尚未全部完成。  
3. 论文初稿已成型，但实验结果章节仍需补充更多对照组与统计分析。  

针对上述问题，整改措施如下：  
1. 固化监控组件配置与部署参数，形成可复用模板，确保每次实验可复现。  
2. 按“静态配置 / HPA-CPU / 多指标 HPA / HPA+VPA”四组路线补齐实验。  
3. 统一结果记录规范，补充时延分位数、吞吐、资源利用率与伸缩次数等关键指标，完善图表与结论对应关系。  
4. 采用周计划推进，先完成多指标与协同实验，再集中完善论文实验与讨论章节。
