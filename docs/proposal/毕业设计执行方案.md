# 基于容器编排的微服务弹性伸缩策略研究与实践执行方案

## 1. 项目定位

本毕业设计面向 Kubernetes 场景下的微服务弹性伸缩问题，构建一套可复现实验平台，用于比较不同伸缩策略在突发负载下的性能、资源利用率、稳定性与成本代理指标。

项目不是传统业务管理系统，而是“微服务弹性伸缩实验与评估系统”。核心交付物包括：

- 可部署的示例微服务 `sample-api`。
- Kubernetes 基础部署、HPA、VPA、Prometheus 与 Prometheus Adapter 配置。
- Locust 压测脚本与自动化实验脚本。
- 原始实验数据、汇总数据、图表和论文/答辩材料。

## 3. 技术路线

1. 使用 Minikube 搭建本地 Kubernetes 实验环境。
2. 将 `sample-api` 打包为容器镜像并部署到集群。
3. 通过 metrics-server 提供 CPU/Memory 资源指标。
4. 通过 Prometheus 采集应用 `/metrics` 暴露的请求计数指标。
5. 通过 Prometheus Adapter 将 QPS 暴露为 Kubernetes 自定义指标。
6. 设计四组策略进行对照实验：`static`、`hpa_cpu`、`hpa_multi`、`hpa_vpa`。
7. 使用 Locust 构造突发负载，采集性能、资源和伸缩行为数据。
8. 使用 Python 脚本汇总数据并生成论文图表。

## 4. 实验评价指标

- 性能指标：P95、P99、平均响应时间、QPS。
- 稳定性指标：失败率、伸缩事件数、振荡事件数。
- 资源指标：CPU 利用率、内存利用率。
- 成本代理指标：按 CPU/Memory 使用量加权估算的 `cost_per_1k_req`。

成本代理指标只用于策略横向比较，不代表真实云厂商账单。

## 6. 验收标准

项目验收时应能够说明并展示：

1. 仓库结构完整，README 能指导复现实验流程。
2. `sample-api` 能被容器化部署到 Kubernetes。
3. HPA/VPA/Prometheus/Adapter 配置文件与实验目标对应。
4. `scripts/run_experiments.ps1` 能串联部署、压测、采集、归档、汇总和出图。
5. 正式实验结果包含 4 组策略各 3 次运行记录。
6. 论文结论能由 raw CSV、summary CSV、figures 和 manifest 共同支撑。
