## 3. 正式实验状态

正式实验已经完成：

- 场景：`burst`
- 策略：`static`、`hpa_cpu`、`hpa_multi`、`hpa_vpa`
- 重复次数：每组 3 次
- 总记录数：12 条
- 数据质量：`top_sample_fallback=0`，`hpa_vpa` 组 `vpa_valid=1`

主要趋势：

- `hpa_vpa` 组 P95 延迟最低、QPS 最高、失败率最低。
- `static` 组在突发负载下失败率和成本代理指标较高。
- 多指标策略比单 CPU 策略更能反映真实业务压力。
