# 基于容器编排的微服务弹性伸缩策略研究与实践（中期版初稿）

## 摘要

随着云原生架构的发展，微服务系统在可扩展性和可维护性方面具有明显优势，但同时也带来了资源管理复杂、负载波动响应困难等问题。本文围绕 Kubernetes 场景下微服务弹性伸缩展开研究，重点针对传统单指标伸缩策略在响应滞后、资源利用率不高和伸缩稳定性不足方面的局限，构建并验证可复用的实验平台与伸缩策略实现路径。  
当前阶段已完成 Kubernetes 实验环境搭建、基础服务部署、HPA（CPU）策略接入、监控链路打通与压测流程自动化。通过两轮压测获取了初步结果，验证了系统在高并发场景下的稳定性与可伸缩性。后续将进一步开展多指标 HPA、HPA-VPA 协同与对比实验，并完善论文中结果分析与优化建议部分。

关键词：微服务；Kubernetes；弹性伸缩；HPA；VPA；云原生

## 1 绪论

### 1.1 研究背景

在业务流量持续波动和成本约束持续增强的背景下，企业对于“按需分配计算资源”的需求越来越高。静态资源配置往往导致高峰期资源不足、低峰期资源浪费，难以兼顾性能与成本。Kubernetes 提供了 HPA、VPA 等自动伸缩机制，为微服务弹性治理提供了基础能力，但在工程实践中仍面临指标单一、策略冲突与参数调优困难等问题。

### 1.2 研究意义

本课题的研究意义体现在三个方面：  
1. 工程价值：形成可复现、可迁移的伸缩实践流程。  
2. 应用价值：在保障服务质量的同时提升资源利用率，降低运维成本。  
3. 学术价值：为多指标伸缩和协同控制提供可量化的实验依据。

### 1.3 研究内容与目标

本文目标是构建微服务弹性伸缩实验平台，逐步完成以下任务：  
1. 完成基础 HPA 验证与基线数据采集。  
2. 实现多指标 HPA（CPU/内存/业务指标）策略。  
3. 实现 HPA 与 VPA 协同控制并进行稳定性评估。  
4. 形成对照实验与结论，输出可复用配置模板。

## 2 相关技术与研究现状

### 2.1 容器编排与自动伸缩机制

Kubernetes 是主流容器编排平台，HPA 通过副本数扩缩实现水平伸缩，VPA 通过资源请求推荐与调整实现垂直伸缩。二者结合可在不同负载阶段提供更细粒度资源调度能力。

### 2.2 国内外研究现状

现有研究主要聚焦三类路径：  
1. 响应式伸缩（基于 CPU/内存阈值）。  
2. 多指标融合伸缩（引入 QPS、延迟等业务指标）。  
3. 预测式伸缩（引入时序模型实现前瞻扩容）。  
现阶段挑战主要集中于指标协同、策略稳定性和工程可落地性。

## 4 中期实验与结果分析

### 4.1 实验设计

已完成两轮基础实验：  
1. 基线压测：中等并发负载，验证服务稳定性。  
2. 高并发压测：提升用户数与增长速率，观察 HPA 行为。

### 4.2 阶段性结果

高并发实验统计结果（Aggregated）：  
- 请求总数：119141  
- 失败数：0  
- 平均响应时间：77.03 ms  
- P95：130 ms  
- P99：210 ms  

实验中观察到 Deployment 副本从 2 扩容到 3，说明当前 HPA 已具备基础自动扩缩容能力。

### 4.3 阶段结论

当前系统已具备中期检查要求的“可运行、可测试、可出数”能力，但对照组尚不完整。下一阶段重点应从“能跑通”转向“能对比、能分析、能形成结论”。
